# MessageCoach クイックスタート

あなた用の起動・使い方ガイドです。

---

## 起動方法

### 1. 環境準備（初回のみ）

```bash
cd C:\Users\aki51\Projects\message-analyzer
npm install
```

### 2. APIキー設定（初回のみ）

`.env.local` ファイルを作成して、OpenAI APIキーを設定：

```
OPENAI_API_KEY=sk-あなたのキー
```

### 3. 起動

```bash
npm run dev
```

ブラウザで http://localhost:3000 を開く

### 4. 停止

ターミナルで `Ctrl + C`

## 基本の使い方

### いきなり添削（ゲスト添削）

相手を登録しなくても添削できます：

1. http://localhost:3000/submit を開く
2. 直近の会話を貼り付け（10〜30通でOK）
3. 送信予定文を入力
4. 目的・トーンを選ぶ
5. 「添削する」をクリック

### 相手を登録して効率化

登録すると次回から直近ログだけでOK：

1. http://localhost:3000/partners → 「新規追加」
2. 名前を入力 + 会話ログ全文を貼り付け
3. 「保存」すると次回から「直近ログだけ」で添削できる

### スクショ読取（OCR）

テキストコピーできないアプリ（Tinderなど）の場合：

1. 添削画面で「スクショ」タブを選択
2. スクリーンショットをアップロード（最大3枚）
3. 「読み取る」をクリック
4. 結果を確認・修正して使用

## 困ったとき

| 症状 | 対処 |
|------|------|
| 起動しない | `npm install` を再実行 |
| APIエラー（insufficient_quota） | OpenAIの残高を確認・チャージ |
| APIエラー（401） | `.env.local` のキーを確認 |
| 相手が消えた | localStorageがクリアされた（ブラウザ設定） |
| ビルドエラー | `npm run build` で詳細確認 |

## 本番ビルド（デプロイ時）

```bash
npm run build   # ビルド
npm start       # 本番起動
```

---

## データの保存場所

- **相手プロファイル**: ブラウザのlocalStorage（サーバーには保存されない）
- **会話ログ**: 保存されない（添削時のみ使用）
- **スクリーンショット**: 保存されない（読み取り処理のみ）

プライバシー重視の設計なので、ブラウザを変えると相手データは引き継がれません。
