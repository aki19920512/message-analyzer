# メッセージ解析

AIがメッセージを解析し、改善提案を行うWebアプリケーションです。

## 機能

- **相手プロファイル管理**: 会話ログからAIが相手の特性を分析・保存
- **ワンタップ添削**: プロファイル + 直近ログだけで高速に添削
- **スクショ読み取り（OCR）**: テキストコピーできないアプリでもスクリーンショットから会話を抽出
- **スコア評価**: 温度感、圧力リスク、誠実さ、明確さ、スタイル適合の5軸評価
- **3パターンの改善提案**: 丁寧/カジュアル/ユーモアの3案を提示
- **ダッシュボード**: 相手一覧から状態を一目で確認

## セットアップ

### 必要な環境

- Node.js 18以上
- npm

### インストール

```bash
cd message-analyzer
npm install
```

### 環境変数の設定

`.env.example` をコピーして `.env.local` を作成し、OpenAI APIキーを設定:

```bash
cp .env.example .env.local
```

`.env.local` を編集:

```
OPENAI_API_KEY=sk-your-actual-api-key
```

### 開発サーバーの起動

```bash
npm run dev
```

ブラウザで http://localhost:3000 を開く

## 使い方

### ゲスト添削（相手未登録でもOK）

初回は相手を登録せずに添削を試せます：

1. `/submit` にアクセス
2. 直近の会話をコピペ（10〜30通でOK）
3. 送信予定文を入力
4. 目的とトーンを選択
5. 「添削する」をクリック

慣れたら相手を保存すると、次回から直近ログだけで添削できます。

### Step 1: 相手を登録する

1. `/partners`（相手一覧）→「新規追加」をクリック
2. 相手の名前（ニックネームでOK）を入力
3. LINEやメッセージの会話ログを貼り付け（最大20,000文字）
4. 「プロファイルを作成して保存」をクリック
5. AIが相手の特性を分析してプロファイルを作成

### Step 2: メッセージを添削する

1. `/partners`（相手一覧）→ 相手カードの「添削する」をクリック
2. **直近の会話**を入力（3つの方法から選べます）:
   - **貼り付け**: LINEなどからコピペ（20〜30行程度でOK）
   - **スクショ読取**: テキストコピーできない場合、スクリーンショットからOCR
   - **全文で更新**: 久々の場合、全文からプロファイル更新
3. **送信予定文**を入力
4. **目的**と**トーン**を選択
5. 「解析する」をクリック
6. 結果を確認、改善案をコピー

### スクショ読み取り（OCR）

テキストコピーできないアプリの場合、スクリーンショットから会話を読み取れます。

1. 添削画面で「スクショ読取」タブを選択
2. スクリーンショットをアップロード（PNG/JPG/WEBP、最大3枚・各5MB）
3. 「読み取る（OCR）」をクリック
4. 読み取り結果を確認・修正（誤読がある場合は手動で直してください）
5. 「このテキストを直近ログに使う」をクリック

**注意:**
- スクリーンショットは読み取り処理のためだけにサーバーに送信され、保存されません
- 固有名詞（本名など）が含まれる場合は、読み取り後に伏せてください
- 読み取り精度は100%ではないため、必ず結果を確認してください
- OpenAI Vision API（gpt-4o-mini）を使用します（追加のAPIキーは不要）

### Step 3: 繰り返す

- ダッシュボードで相手ごとの直近スコアを確認
- ワンタップで添削画面へ移動

### 使い分けガイド

| シーン | やること | 入力するもの |
|--------|----------|------------|
| **普段の添削** | 「貼り付け」タブ | 直近の会話（コピペでOK、10〜30通で十分） |
| **コピペできない場合** | 「スクショ読取」タブ | スクリーンショット画像（最大3枚） |
| **久々・状況が変わった** | 「全文で更新」タブ | 会話ログ全文（初回と同じ要領） |

- **直近ログ**: LINEなどから最後の10〜30通をコピペするだけでOKです
- **スクショ読取**: テキストコピーできないアプリの場合、スクリーンショットからAIが会話テキストを読み取ります
- **全文ログ**: 初回登録のときと、久しぶりに相談するとき（相手の状況が変わっていそうなとき）だけ必要です
- プロファイル更新後は、また直近ログだけで添削できます

## プライバシー

- **会話ログは保存しません** (localStorageにもサーバーにも)
- **スクリーンショット画像は保存しません** (読み取り処理のみに使用し、サーバーやローカルに保存しません)
- 保存されるのはAI生成のプロファイル（テキスト要約）とスコア（数値）のみ
- ログに入力全文を出力しません
- 解析・OCRはOpenAI APIを通じて行われます

## 制約

- 初回登録用会話ログ: 最大20,000文字
- 添削用直近ログ: 最大6,000文字
- プロファイル: 最大2,000文字
- 送信予定文: 最大400文字
- スクリーンショット: 最大3枚、1枚あたり最大5MB（PNG/JPG/WEBP）

## スモークテスト（KB統合）

KB（ナレッジベース）が正常に動作していることを確認するテストです。

### A. KBありで参照ルールが表示される

1. `data/rule_cards.jsonl` が存在することを確認
2. `npm run dev` で開発サーバーを起動
3. `/submit?mode=analyze` にアクセス
4. 相手を選択（または新規にプロファイル入力）
5. 直近の会話と送信予定文を入力
6. 目的を「謝る」などKBに存在するタグを選択
7. 「添削する」をクリック
8. 結果画面に「参照ルール」セクションが表示されることを確認

### B. KB不在でもgraceful degradation

1. `data/rule_cards.jsonl` を一時的にリネーム:
   ```bash
   mv data/rule_cards.jsonl data/rule_cards.jsonl.bak
   ```
2. サーバーを再起動（キャッシュクリアのため）
3. 同様に添削を実行
4. 解析が成功し、「参照ルール」セクションが表示されない（または空）ことを確認
5. ファイルを元に戻す:
   ```bash
   mv data/rule_cards.jsonl.bak data/rule_cards.jsonl
   ```

### C. 壊れた行のスキップ確認

1. `data/rule_cards.jsonl` の末尾に不正なJSONを追加:
   ```bash
   echo "this is not valid json" >> data/rule_cards.jsonl
   ```
2. サーバーを再起動
3. 添削を実行
4. 解析が成功することを確認
5. サーバーログに `[KB] Skipped 1 malformed line(s)` が出力されることを確認
6. 不正な行を削除して元に戻す
